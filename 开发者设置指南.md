# 开发者设置指南 - Supabase 云端同步

## 📋 准备工作

- ✅ 一个邮箱账号（用于注册 Supabase）
- ✅ 5-10 分钟时间
- ✅ 代码编辑器（VS Code 等）

---

## 🚀 第一步：创建 Supabase 账号和项目

### 1.1 注册 Supabase 账号

1. 打开浏览器，访问：**https://supabase.com**
2. 点击右上角的 **"Start your project"** 或 **"Sign up"**
3. 选择注册方式（推荐使用 GitHub，也可以使用 Google 或邮箱）
4. 完成注册流程

### 1.2 创建新项目

1. 登录后，点击 **"New Project"** 按钮
2. 填写项目信息：

   **Organization（组织）**：
   - 如果是第一次使用，点击 **"New Organization"** 创建一个
   - 组织名称可以填写：`My Projects` 或任意名称
   - 点击 **"Create organization"**

   **Project Name（项目名称）**：
   - 填写：`italian-vocabulary`（或任意名称）

   **Database Password（数据库密码）**：
   - ⚠️ **重要**：设置一个强密码（至少12位，包含大小写字母、数字）
   - 📝 **请记住这个密码**，以后可能需要用到
   - 建议保存到密码管理器

   **Region（区域）**：
   - 选择离您最近的区域
   - 中国用户建议选择：`Southeast Asia (Singapore)` 或 `Northeast Asia (Tokyo)`

3. 勾选同意条款
4. 点击 **"Create new project"**
5. ⏳ 等待项目创建完成（通常需要 1-2 分钟）

---

## 📊 第二步：创建数据表

### 2.1 进入表编辑器

1. 项目创建完成后，会自动进入项目控制台
2. 在左侧菜单栏，找到并点击 **"Table Editor"**（表格编辑器）

### 2.2 创建新表

1. 点击 **"Create a new table"** 按钮
2. 填写表信息：

   **Name（表名）**：
   - 填写：`user_data`
   - ⚠️ **必须完全一致**，区分大小写

   **Description（描述）**（可选）：
   - 填写：`User data for Italian vocabulary app`

3. 点击 **"Save"** 保存表

### 2.3 添加列（Columns）

表创建后，会自动打开表编辑器。现在需要添加列：

#### 列 1：username

1. 点击 **"Add column"** 或 **"New column"**
2. 填写列信息：
   - **Name**: `username`
   - **Type**: 选择 `text`
   - **Is Primary Key**: ✅ **勾选**（重要！）
   - **Is Nullable**: ❌ **不勾选**（必填）
3. 点击 **"Save"** 或 **"Add"**

#### 列 2：data

1. 再次点击 **"Add column"**
2. 填写列信息：
   - **Name**: `data`
   - **Type**: 选择 `jsonb`（在类型下拉菜单中，可能需要滚动查找）
   - **Is Primary Key**: ❌ **不勾选**
   - **Is Nullable**: ❌ **不勾选**（必填）
3. 点击 **"Save"** 或 **"Add"**

#### 列 3：updated_at

1. 再次点击 **"Add column"**
2. 填写列信息：
   - **Name**: `updated_at`
   - **Type**: 选择 `timestamptz`（timestamp with time zone）
   - **Is Primary Key**: ❌ **不勾选**
   - **Is Nullable**: ❌ **不勾选**（必填）
   - **Default Value**: 填写 `now()`（这样会自动设置当前时间）
3. 点击 **"Save"** 或 **"Add"**

### 2.4 确认表结构

最终表结构应该是：

| 列名 | 类型 | 主键 | 必填 | 默认值 |
|------|------|------|------|--------|
| username | text | ✅ | ✅ | - |
| data | jsonb | ❌ | ✅ | - |
| updated_at | timestamptz | ❌ | ✅ | now() |

## 🔑 第三步：获取 API 密钥

### 3.1 进入 API 设置

1. 在左侧菜单栏，找到并点击 **"Settings"**（设置，⚙️ 图标）
2. 在设置菜单中，点击 **"API"**

### 3.2 复制配置信息

在 API 设置页面，您会看到以下信息：

#### Project URL（项目URL）

- 位置：在页面顶部，**"Project URL"** 部分
- 格式：`https://xxxxxxxxxxxxx.supabase.co`
- 📋 **复制完整的 URL**（包括 `https://`）

#### API Keys（API 密钥）

- 位置：在 **"Project API keys"** 部分
- 找到 **"anon public"** 这一行
- 📋 **复制完整的 key**（很长的一串字符，以 `eyJ` 开头）

⚠️ **重要提示**：
- ✅ 使用 **"anon public"** key（可以在浏览器中使用）
- ❌ **不要使用** "service_role" key（这是私密密钥，只能在服务器端使用）

## 💻 第四步：配置代码

### 4.1 打开配置文件

1. 在您的项目中，打开文件：`js/supabase-storage-manager.js`
2. 找到第 6-7 行：

```javascript
this.supabaseUrl = 'YOUR_SUPABASE_URL'; // 例如：https://xxxxx.supabase.co
this.supabaseKey = 'YOUR_SUPABASE_ANON_KEY'; // 匿名公钥，可以在浏览器中使用
```

### 4.2 替换配置

将刚才复制的信息替换到代码中：

```javascript
// 替换前
this.supabaseUrl = 'YOUR_SUPABASE_URL';
this.supabaseKey = 'YOUR_SUPABASE_ANON_KEY';

// 替换后（示例）
this.supabaseUrl = 'https://abcdefghijklmnop.supabase.co';
this.supabaseKey = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImFiY2RlZmdoaWprbG1ub3AiLCJyb2xlIjoiYW5vbiIsImlhdCI6MTY0NzI5NDU2MywiZXhwIjoxOTYyODcwNTYzfQ.xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx';
```

⚠️ **注意事项**：
- URL 和 Key 都要用单引号或双引号包裹
- 确保没有多余的空格
- 确保 URL 以 `https://` 开头，以 `.supabase.co` 结尾

### 4.3 保存文件

保存 `js/supabase-storage-manager.js` 文件

## 🔒 第五步：设置表权限（重要！）

这一步非常重要，如果不设置权限，用户将无法访问数据库。

### 5.1 打开 SQL 编辑器

1. 在 Supabase 控制台左侧菜单，点击 **"SQL Editor"**（SQL 编辑器）
2. 点击 **"New query"**（新建查询）

### 5.2 执行 SQL 语句

在 SQL 编辑器中，粘贴以下 SQL 代码：

```sql
-- 启用行级安全策略
ALTER TABLE user_data ENABLE ROW LEVEL SECURITY;

-- 设置username为唯一约束（用于UPSERT操作）
CREATE UNIQUE INDEX IF NOT EXISTS user_data_username_idx ON user_data(username);

-- 允许匿名用户插入和更新数据
CREATE POLICY "Allow anonymous insert and update"
ON user_data
FOR ALL
TO anon
USING (true)
WITH CHECK (true);
```

### 5.3 执行 SQL

1. 点击 SQL 编辑器右下角的 **"Run"** 按钮（或按 `Ctrl+Enter` / `Cmd+Enter`）
2. 等待执行完成
3. 如果看到成功消息，说明执行成功

⚠️ **如果出现错误**：
- 如果提示 `user_data_username_idx` 已存在，可以忽略（说明索引已经创建）
- 如果提示其他错误，请检查表名是否正确（应该是 `user_data`）

## 🔧 常见问题排查

### 问题1：同步失败，控制台显示 401 错误

**原因**：API 密钥配置错误或权限未设置

**解决方法**：
1. 检查 `supabase-storage-manager.js` 中的 API 密钥是否正确
2. 确认使用的是 `anon public` key，不是 `service_role` key
3. 重新执行第五步的 SQL 权限设置

### 问题2：同步失败，控制台显示 404 错误

**原因**：表名错误或 URL 配置错误

**解决方法**：
1. 检查表名是否为 `user_data`（完全一致）
2. 检查 `supabaseUrl` 是否正确（应该以 `.supabase.co` 结尾）

### 问题3：同步失败，控制台显示 409 错误

**原因**：唯一约束冲突（正常情况，代码会自动处理）

**解决方法**：
- 这个错误通常会被自动处理，如果持续出现，检查 SQL 索引是否创建成功

### 问题4：数据没有同步到云端

**解决方法**：
1. 打开浏览器控制台（F12）
2. 查看 Network（网络）标签，检查是否有请求发送
3. 检查请求的 URL 和响应状态
4. 确认网络连接正常
