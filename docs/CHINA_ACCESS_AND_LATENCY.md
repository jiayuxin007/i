# 中国用户访问慢 / 加载久 问题说明

## 现象

- 在意大利开发/部署，中国用户访问时**响应很慢**，尤其「生成单词测验选项」和「结果里的中文释义」要 **1 分钟以上**。

## 原因（网络/接口在中国不可用或极慢）

和「网络问题」直接相关：页面用到的**国外接口在中国大陆被墙或极慢**，请求会长时间挂起直到超时。

### 1. 单词测验「生成选项」阶段

- **调用**：`js/llm-api.js` → `generateWithOpenAI()`  
- **请求**：`https://api.openai.com/v1/chat/completions`（OpenAI）
- **在中国**：OpenAI 无法直连，请求会一直转圈或很久才超时，导致「生成选项中...」卡住。

### 2. 选项与结果里的「中文释义」

- **调用**：`js/translation-helper.js` → `getChineseTranslation()` / `batchTranslate()`
- **请求**：`https://translate.googleapis.com/translate_a/single?...`（Google 翻译）
- **在中国**：Google 服务被墙，该请求同样会挂起或超时，导致选项/结果加载很久。

### 3. 其他可能慢但非主因

- 云端同步：`api.github.com`、Supabase 等在中国有时也慢，但一般不会单独导致「1 分钟」级卡顿。
- 词汇数据：`source_italiano/italian.json` 同源加载，不受墙影响。

**结论**：主要瓶颈是 **OpenAI** 和 **Google 翻译** 在中国不可用或极慢，导致测验选项和中文释义阶段长时间等待。

## 已做优化（减少等待时间）

1. **Google 翻译**（`translation-helper.js`）  
   - 为请求增加了 **8 秒超时**（AbortController）。  
   - 超时后立刻失败，返回 `null`，页面只显示英文或「—」，不再等 1 分钟。

2. **OpenAI 生成选项**（`llm-api.js`）  
   - 为请求增加了 **15 秒超时**。  
   - 超时或失败后走**本地备用逻辑**：从同等级词库随机选 3 个作干扰项，不依赖 OpenAI，选项仍可正常出现。

这样在中国即使用户不翻墙，最多等约 8～15 秒就会失败并降级，而不会卡在「生成选项中...」或「翻译中...」超过 1 分钟。

## 可选方案（改善中国用户体验）

1. **不配置 OpenAI API Key**  
   - 测验始终用本地词库生成干扰项，不请求 OpenAI，国内加载会更快、更稳定。

2. **翻译接口**  
   - 国内用户可考虑换用**国内可访问的翻译 API**（如百度、有道等），在 `translation-helper.js` 里对国内环境做分支或可配置的翻译源。  
   - 或由后端代理 Google/其他翻译，前端只请求自己的后端，避免直连被墙域名。

3. **预置中文释义**  
   - 在 `italian.json`（或单独词表）里为单词预填中文释义，国内不请求翻译接口也能显示中文。

4. **部署与 CDN**  
   - 若站点部署在海外，可把静态资源（含 `italian.json`）放到国内可访问的 CDN，减少首屏和词库加载延迟；接口问题仍需要上述 1～3 之一配合解决。

---

**总结**：卡顿主要来自 **OpenAI** 和 **Google 翻译** 在中国不可用或极慢；已为这两处请求加超时并做了降级，国内用户最长等待时间已从 1 分钟以上降到约 8～15 秒，并保证测验选项和页面仍可正常使用。
