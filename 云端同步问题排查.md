# 云端同步问题排查指南

## 🔍 快速诊断

如果云端同步不生效，请按照以下步骤排查：

---

## 第一步：检查浏览器控制台错误

### 1.1 打开开发者工具

1. **按 `F12`** 打开浏览器开发者工具
2. **点击 "Console"**（控制台）标签
3. **注册或登录一个账号**
4. **查看是否有红色错误信息**

### 1.2 常见错误及解决方法

#### 错误：`云端同步未配置`
**原因**：API密钥配置错误  
**解决**：检查 `js/supabase-storage-manager.js` 中的配置是否正确

#### 错误：`401 Unauthorized`
**原因**：API密钥错误或权限未设置  
**解决**：
1. 检查API密钥是否正确（anon public key）
2. 确认已执行SQL权限设置（见开发者设置指南.md）

#### 错误：`404 Not Found`
**原因**：表名错误或URL错误  
**解决**：
1. 检查表名是否为 `user_data`（完全一致）
2. 检查Supabase URL是否正确

#### 错误：`CORS error` 或 `网络错误`
**原因**：网络连接问题或CORS配置  
**解决**：检查网络连接，确保可以访问Supabase

## 第三步：检查Supabase数据库

### 3.1 查看表是否存在

1. **登录Supabase控制台**：https://supabase.com
2. **进入项目**
3. **点击左侧 "Table Editor"**
4. **检查是否有 `user_data` 表**

### 3.2 检查表结构

确保表有以下列：
- `username` (text, 主键)
- `data` (jsonb)
- `updated_at` (timestamptz)

### 3.3 检查表权限

1. **点击左侧 "SQL Editor"**
2. **执行以下SQL检查权限**：

```sql
SELECT * FROM pg_policies WHERE tablename = 'user_data';
```

**如果返回空**：说明权限未设置，需要执行权限设置SQL（见开发者设置指南.md）

### 3.4 检查数据是否已存储

1. **在 Table Editor 中打开 `user_data` 表**
2. **查看是否有数据**（应该有一条 `username` 为 `_all_users_` 的记录）

**如果没有数据**：说明同步失败，查看控制台错误

## 第五步：常见问题解决方案

### 问题1：权限错误（401）

**症状**：控制台显示 401 Unauthorized

**解决方法**：
1. 确认使用的是 `anon public` key，不是 `service_role` key
2. 在Supabase控制台执行以下SQL：

```sql
-- 启用行级安全策略
ALTER TABLE user_data ENABLE ROW LEVEL SECURITY;

-- 删除旧策略（如果存在）
DROP POLICY IF EXISTS "Allow anonymous insert and update" ON user_data;

-- 创建新策略
CREATE POLICY "Allow anonymous insert and update"
ON user_data
FOR ALL
TO anon
USING (true)
WITH CHECK (true);
```

3. 刷新页面，重新测试

### 问题2：表不存在（404）

**症状**：控制台显示 404 Not Found

**解决方法**：
1. 确认表名为 `user_data`（完全一致，区分大小写）
2. 在Supabase控制台创建表（见开发者设置指南.md）

### 问题3：唯一约束冲突（409）

**症状**：控制台显示 409 Conflict

**解决方法**：
这个错误通常会被自动处理，但如果持续出现：
1. 在Supabase控制台执行：

```sql
-- 确保username是唯一约束
CREATE UNIQUE INDEX IF NOT EXISTS user_data_username_idx ON user_data(username);
```

### 问题4：数据没有同步

**症状**：注册/登录后，Supabase中没有数据

**解决方法**：
1. 打开浏览器控制台（F12）
2. 查看 Network（网络）标签
3. 注册或登录账号
4. 查找对 `supabase.co` 的请求
5. 检查请求状态和响应

**如果请求发送但失败**：
- 查看响应内容，找出错误原因
- 通常是权限或配置问题

**如果请求没有发送**：
- 检查 `isConfigured()` 是否返回 true
- 检查代码是否有错误

## 第六步：启用详细日志

在 `js/supabase-storage-manager.js` 中添加更多日志：

找到 `syncAllDataToCloud` 方法，在开头添加：

```javascript
async syncAllDataToCloud() {
    console.log('开始同步数据到云端...');
    console.log('配置状态:', this.isConfigured());
    
    if (!this.isConfigured()) {
        console.error('云端同步未配置！');
        return { success: false, message: '云端同步未配置' };
    }
    // ... 其余代码
}
```

这样可以在控制台看到更详细的同步过程。

## ✅ 验证清单

完成排查后，确认：

- [ ] Supabase配置正确（URL和Key都对）
- [ ] `isConfigured()` 返回 `true`
- [ ] 表 `user_data` 存在且结构正确
- [ ] 表权限已设置（RLS已启用，策略已创建）
- [ ] 浏览器控制台没有错误
- [ ] 网络请求成功发送到Supabase
- [ ] Supabase数据库中有数据（username为`_all_users_`的记录）

如果所有项都打勾，但同步还是不工作，可能是代码逻辑问题，需要进一步检查。
